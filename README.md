      所有保存在GIT数据库中的东西都是用哈希值做索引的，而不是靠文件名。

      GIT只有三种状态：已提交(committed)，已修改(modified)和已暂存(staged).
      第一：已提交表示该文件已经被安全地保存做本地数据库中了。
      第二：已修改表示修改了某个文件，但还没有提交保存。
      第三：已暂存表示已修改的文件放在下次提交时要保存的清单中。
      （提交到服务端：应该是提交哪些存在本地数据库中的文件）
      它对应的是三个工作区域：本地数据目录，工作目录，暂存区域。

每个项目都有一个GIT目录，他是GIT用来保存元数据和对象数据的地方，该目录非常重要，每次克隆镜像仓库的时候，实际拷贝的就是这个目录里的数据。

从项目中取出某个版本的所有文件和目录，用已开始后续工作的叫做工作目录。这些文件实际上都是从GIT目录中的压缩对象数据库提取出来的。接下来就
可以做工作目录中队文件进行编辑。

所谓的暂存区域只不过是个简单的文件，一般都放在GIT目录中，有时候人们会把这个文件叫做索引文件，不过标准说法还是叫暂存区域。

记录每次更新到仓库：
工作目录的所有文件都只有两个状态：已跟踪  与  未跟踪   初次克隆某个仓库时，工作目录中的所有文件都属于已跟踪文件。

只要做“changes to be committed”这行下面的，就说明是已暂存状态。如果此时提交，那么该文件此时此刻的版本将被留存做历史记录中。

git add后可以接要跟踪的文件或者目录的路径：如果是目录的话，就说明要递归跟踪所有该目录下的文件。
git add:这是个多功能命令，根据目标文件的状态不同，此命令的效果也不同：
        可以用它开始跟踪新文件，把已跟踪的文件放做暂存区。
        用于合并是把有冲突的文件标记为已解决状态等。

只要出现“Changed but not updated”这行下面，就说明已跟踪文件的内容发生了变化，但还没有放入暂存区。要暂存这次更新。
需要运行git add命令。
如果只是简单地从工作目录总手工删除文件，运行git status时就会做"Changed but not updated"部分（也就是_未暂存_清单）：

忽略某些文件：文件 .gitignore 的格式规范如下：
第一：所有空行或者注释符号 # 开头的行都会被git忽略。

git status 显示的比较简单，仅仅是列出了修改过的文件。如果要查看修改了什么地方。可以用git diff命令。
现在git status能够回答我们两个问题：
第一：当前做的那些更新没有暂存?
第二：有哪些更新几经暂存起来准备好了下次提交？git diff会使用文件补丁的格式显示添加和删除的行。

提交更新： 暂存区域已经准备妥当可以提交时。一定要确认还有修改过的或新建的文件没有git add过，否则提交的时候不会记录这些
没暂存起来的变化。所以，每次准备提交前，先用git status看下。是不是都暂存起来了，然后再运行提交命令 git commit:

尽管使用暂存区域的方式可以精心准备要提交的细节，但有时候这恶做略显繁琐。git提供了一个跳过使用暂存区域的
方式，只要做提交的时候，给git commit加上 -a 选项， git就会自动把所有已经跟踪的文件暂存起来一并提交，从而跳过git add步骤

移除文件：
要从git中移除某个文件，就必须要从已跟踪文件清单中移除（确切的说，是从暂存区域移除）。然后提交。并连带从工作目录中删除指定
文件，这样以后就不会出现做未跟踪文件清单中了。

移动文件：只有放在缓存区域里的文件，才可以操作移动文件的命令： git mv fromName toName

撤销操纵：
任何时候，你都有可能需要撤销刚才所做的某些操作，并不总是可以撤销所有的操作了，所以还是有些小心的。一旦失误，就有可能
失去部分工作成果。

第一：修改最后一次提交
git commit --amend
此命令将使用当前的暂存区域快照提交，如果刚才提交完没有作任何改动，直接运行命令，相当于有机会重新编辑提交说明，而所提交
的文件快照和之前的一样。

如果提交后又添加了文件进入暂存区域，那么
git commit -m "initial commit"
git add forgotten_file
git commit --amend
上面的三条命令最终得到一个提交，第二个提交命令修改了第一个的提交内容。

远程仓库的使用：
要参与任何换一个GIT项目的协作。必须要了解该如何管理远程仓库。

查看当前的远程库：git remote 命令 origin是git默认使用这个名字来标识你所克隆的原始仓库。git remote -v

添加一个新的远程仓库，可以指定一个简单的名字，以便将来引用。
git remote add [shortname] [url]

git remote add pb git://github.com/paulboone/ticgit.git
现在可以用字符串PB指代对应的仓库地址了。比如说，要抓取所有paul有的但本地仓库没有的信息，可以运行 git fetch pb

古国克隆了一个仓库，此命令会自动将仓库origin名下，所以 git fetch origin会抓取从你上次克隆以来别人上传到次远程中的所
有更新(或是上次fetch以来别人提交的更新) 。有一点很重要，需要记住，fetch命令只是将远端的数据拉到本地仓库。并不自动
合并到当前工作分支，之有当你确实准备好了，才能手工合并。
